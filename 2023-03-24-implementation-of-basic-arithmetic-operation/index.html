前回 は意気込みだけでしたので、 今回から実装に入ります。 今回の実装範囲は四則演算で、参考書の 電卓レベルの言語の作成 節をやっていきます。

Goal

以下のような四則演算を実行し、 process の exit コード で返すようにします。

(1 + 2) * 3 / (5 - 4)

確認は以下のようなコマンドで exitコード を得ることができます。

$ ./foo
$ echo $?
9

実装

AST で構文の木構造が手にはいるのでこれを左から追う感じで x86 アセンブラ へファイルを書き込む実装をします。 できた x86 アセンブラ ファイルを gcc などの コンパイラ でコンパイルして実行できるファイルを生成します。

AST1

前回でもやったよう に パーサー は parser.gem を利用します。 今回の例題として先述した四則演算のファイルをパースして AST から、 アセンブラ に変換します。 まず AST の結果は以下のようになります。

s(:send,
  s(:begin,
    s(:send,
      s(:begin,
        s(:send,
          s(:int, 1), :+,
          s(:int, 2))), :*,
      s(:int, 3))), :/,
  s(:begin,
    s(:send,
      s(:int, 5), :-,
      s(:int, 4))))

ここで :send や :begin などが Node とよばれるものになります。Node を順に辿ることで実行できるような x86 アセンブラ2 を書くことができます。 :int の型情報や :+ などの演算子があることがわかるとおもいます。今はまだ :send や :begin は実装に必要でないので処理では気にせず進めることにします。

Ruby の実装

AST はその名のとおり木なので順を追っていくだけで実行できます。 Ruby のコードから x86 アセンブラ へはこの抽象構文木から得た命令情報を元に組みます。

require "parser/current"
# デバッグフラグ
debug = !!ARGV.shift
# ソースコードの AST を取得
ast = Parser::CurrentRuby.parse("(1 + 2) * 3 / (5 - 4)")

# x86 アセンブラを書き出し
def precompile(ast, f)
  f.puts("")
  f.puts("")
end

# AST に応じてアセンブラに書きかえる
def _precompile(ast, f)
end

f = File.open("tmp.s", "w")
precompile(ast, f)
f.close
io = IO.popen("gcc -o tmp tmp.s").close
File.delete("tmp.s") unless debug
